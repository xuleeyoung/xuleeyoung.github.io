## 一、背景与动机

### 1.1 推荐系统的大模型化趋势

近年来，随着深度学习技术的快速发展，推荐系统正在经历从传统双塔模型向**大模型化**的深刻变革。以 RankMixer、TokenMixer 为代表的**密集特征交互模型（Dense Feature Interaction Models）** 通过更复杂的特征交叉和表示学习，显著提升了推荐效果。然而，这种效果的提升伴随着**计算成本的大幅增长**。

在工业级推荐系统中，模型的推理流程通常包含以下关键步骤：

1. **用户特征编码**：提取用户画像、行为序列、上下文信息
2. **物品特征编码**：处理候选物品的各类特征
3. **特征交互计算**：执行用户-物品之间的深度特征交叉
4. **排序打分**：基于交互结果输出最终分数

### 1.2 现有方法的局限性

对于**序列推荐模型**（如 DIN、DIEN、BST 等），LONGER 等工作已经成功引入了 **KV Cache 机制**来复用用户侧的 Key-Value 计算。每次推理时，只需为用户计算一次注意力状态，便可跨候选物品共享。

然而，**密集特征交互模型**面临一个根本性的挑战：**用户特征与物品特征深度纠缠**。在 RankMixer 等架构中，Token 的表示是用户信息和物品信息**融合后的产物**，无法简单地分离出"纯用户侧"计算。

这种纠缠带来两个核心问题：

1. **无法复用**：每次更换候选物品，都需要重新计算整个交互路径
2. **线性增长**：推理延迟随候选物品数量线性增长

假设候选集包含 $N$ 个物品，模型推理时间复杂度为 $O(N \cdot C)$，其中 $C$ 是单个物品的计算量。这意味着在排序阶段处理大量候选时，计算资源消耗巨大。

### 1.3 研究目标

本文提出的 **UG-Sep（User-Group Separation）** 框架，旨在解决以下核心问题：

> **如何在保持密集特征交互模型表达能力的同时，实现用户侧计算的高效复用？**

## 二、核心方法

### 2.1 UG-Sep 框架概述

UG-Sep 的核心思想是**将 Token 显式解耦为两个语义不同的子集**：

| Token 类型 | 描述 | 特征 |
|-----------|------|------|
| **U-tokens** | User Tokens | 编码纯用户侧信息（用户画像、行为历史） |
| **G-tokens** | Group Tokens | 包含用户-物品交互信息或物品中心特征 |

这种分离的设计哲学可以概括为：

> **"Compute Only Once"** —— 用户侧的计算只需执行一次，便可跨所有候选物品共享。

### 2.2 架构设计

UG-Sep 的整体架构包含以下几个关键组件：

#### 2.2.1 特征提取与分叉

```
                    ┌─────────────┐
    User Features ──│  Feature    │
    (画像/行为序列)  │  Extractor  │
                    │  (SENet/    │
                    │   DCN)      │
                    └──┬───────┬──┘
                       │       │
              ┌────────┘       └────────┐
              ▼                         ▼
        ┌──────────┐             ┌──────────┐
        │  U-branch │             │  G-branch │
        │ (User)   │             │ (Group)  │
        └──────────┘             └──────────┘
```

**分叉模块（Branching Module）** 可以采用多种架构实现：

- **SENet (Squeeze-and-Excitation Network)**：通过通道注意力机制选择性地增强重要特征
- **DCN (Deep & Cross Network)**：显式学习有限阶特征交叉
- **MLP 层**：通过非线性变换实现特征解耦

#### 2.2.2 Token Mixing 与掩码机制

在 Token Mixing 操作（如注意力机制）中，UG-Sep 引入了**结构化掩码（Structured Masking）**：

```
原始注意力矩阵：              UG-Sep 掩码注意力矩阵：
┌─────────┬─────────┐         ┌─────────┬─────────┐
│ U → U   │ U → G   │         │   ✓     │   ✗     │
├─────────┼─────────┤    →    ├─────────┼─────────┤
│ G → U   │ G → G   │         │   ✗     │   ✓     │
└─────────┴─────────┘         └─────────┴─────────┘
```

掩码策略的语义含义：

- **U-tokens 之间**：允许自由交互，保持用户内部信息的完整性
- **G-tokens 之间**：允许物品信息的充分交叉
- **跨类型交互**：被显式抑制，防止用户信息污染物品侧

这种设计确保了 **U-tokens 跨层保持"纯用户侧"表示**，从而实现计算复用。

### 2.3 信息补偿策略

掩码机制虽然实现了信息解耦，但也可能带来**表达能力损失**。UG-Sep 引入了**信息补偿（Information Compensation）** 机制：

#### 2.3.1 自适应重建

```python
class InformationCompensation(nn.Module):
    def __init__(self, hidden_dim):
        self.projection = nn.Linear(hidden_dim, hidden_dim)
        self.gate = nn.Sigmoid()
        
    def forward(self, u_tokens, g_tokens):
        # 基于 G-tokens 重建被抑制的用户-物品交互
        compensation = self.projection(g_tokens)
        gate = self.gate(u_tokens @ compensation.T)
        return u_tokens + gate * compensation
```

补偿机制的核心思想：**将被抑制的用户-物品交互信息，通过可复用的 G-tokens 自适应地重建回来**。

#### 2.3.2 层级补偿策略

补偿可以在不同层级进行：

1. **Token 级补偿**：为每个 U-token 独立计算补偿量
2. **Layer 级补偿**：在深层统一进行补偿
3. **自适应补偿**：根据任务损失动态调整补偿强度

### 2.4 W8A16 权重量化

在 UG-Sep 大幅减少用户侧 FLOPs 后，系统瓶颈从**计算密集型**转变为**内存带宽受限型**。为此，UG-Sep 进一步引入了 **W8A16 量化策略**：

| 量化类型 | 权重精度 | 激活精度 | 适用场景 |
|---------|---------|---------|---------|
| FP16/BF16 | 16-bit | 16-bit | 基准精度 |
| W8A16 | 8-bit | 16-bit | **UG-Sep 优化** |

**W8A16 的优势**：

- **内存占用减半**：权重从 16-bit 降至 8-bit
- **精度保持**：激活值保持 16-bit，保证计算精度
- **带宽优化**：特别适合内存带宽成为瓶颈的场景

### 3.1 计算复杂度分析

设模型包含 $L$ 层，每层有 $H$ 个隐藏单元。对于 $N$ 个候选物品，推理复杂度对比如下：

$$
\text{FLOPs}_{\text{traditional}} = O(N \cdot L \cdot H^2)
$$

**UG-Sep 模型**：

$$
\text{FLOPs}_{\text{UG-Sep}} = O(\underbrace{L \cdot H^2}_{\text{用户侧（一次性）}} + \underbrace{N \cdot \alpha \cdot L \cdot H^2}_{\text{物品侧}})
$$

其中 $\alpha < 1$ 是 G-tokens 占比。

**加速比**：

$$
\text{Speedup} = \frac{\text{FLOPs}_{\text{traditional}}}{\text{FLOPs}_{\text{UG-Sep}}} = \frac{N}{1 + \alpha \cdot N}

当 $N$ 较大且 $\alpha$ 较小时，加速比趋近于 $\frac{1}{\alpha}$。

## 六、技术洞见

### 6.1 "计算一次"的设计哲学

UG-Sep 体现了**边际成本递减**的设计思想：

> **一次计算，多次复用**

这种哲学在推荐系统中尤为重要，因为：
- 候选物品数量可达数百甚至数千
- 用户请求频率高，时间敏感性强
- 计算资源成本显著影响系统利润

### 6.2 用户-物品解缠的通用意义

UG-Sep 的核心贡献不仅在于效率优化，更在于提出了**用户-物品解缠（User-Item Disentanglement）** 这一通用范式：

- **特征层面**：将用户特征和物品特征显式分离
- **计算层面**：用户侧计算与物品侧计算独立
- **存储层面**：可复用计算结果可被持久化缓存

这种解缠思维可以推广到其他领域：
- 多任务学习中的任务解耦
- 跨域推荐中的域解耦
- 冷启动场景中的先验-后验解耦

### 6.3 掩码+补偿的优雅设计

掩码机制引入了**结构化先验**，而补偿机制提供了**自适应调整能力**。两者结合：

- **掩码**：提供强归纳偏置，保证解缠性质
- **补偿**：提供灵活性，减少信息损失

这种"约束 + 自适应"的范式在深度学习中具有广泛的应用价值。
